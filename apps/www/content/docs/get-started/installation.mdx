## Requirements

- React **19** (peer dep)
- Tailwind CSS **v4** (peer dep)
- A bundler — Vite, Next.js, Remix, TanStack Start, Bun.serve with HTML imports, etc.

## Install

```sh
bun add @mxv/ui
```

Or with npm / pnpm / yarn — mxv-ui is plain ESM, package manager doesn't matter.

## Set up your stylesheet

In your global CSS (e.g. `src/styles.css`), import Tailwind, point it at
`@mxv/ui`'s shipped JS so it can find the class names used inside the
components, then import mxv-ui's tokens and base layer:

```css
@import "tailwindcss";
@source "../node_modules/@mxv/ui/dist";
@import "@mxv/ui/globals.css";
```

Why each line matters:

- **`@import "tailwindcss"`** — Tailwind v4's entry. mxv-ui doesn't ship
  Tailwind for you; your app owns the Tailwind version.
- **`@source "..."`** — Tailwind v4 only scans files inside your app by default.
  `node_modules/@mxv/ui` is invisible to it without this directive, so utility
  classes used inside mxv-ui's components (`bg-accent`, `text-fg-muted`, …)
  never get generated and components render unstyled. Adjust the relative path
  to wherever your stylesheet sits relative to `node_modules`.
- **`@import "@mxv/ui/globals.css"`** — pulls in the `@theme` block (color /
  radius tokens exposed as Tailwind utilities like `bg-bg`, `rounded-lg`),
  the base CSS variables (`--bg`, `--fg`, `--accent`, …), and the keyframes
  used by Dialog overlay / content animations.

## Use a component

```tsx
import { Button } from "@mxv/ui"

export default function Page() {
  return <Button tone="primary">Click me</Button>
}
```

## Dark mode

mxv-ui ships a `.dark` class variant. Toggle it on `<html>` or `<body>`:

```tsx
<html className={isDark ? "dark" : ""}>
```

All tokens flip automatically. There's no provider component — it's just a class.
