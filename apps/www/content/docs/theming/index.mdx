mxv-ui has no `<MxvProvider>`, no theme config object. Theming is just CSS
variables — override them in your stylesheet and every component picks up the
change.

## The full token surface

```css
:root {
  /* Surfaces */
  --bg:           oklch(0.985 0.004 80);
  --bg-subtle:    oklch(0.965 0.004 80);
  --bg-muted:     oklch(0.93 0.005 80);

  /* Text */
  --fg:           oklch(0.22 0.02 260);
  --fg-muted:     oklch(0.5 0.02 260);
  --fg-subtle:    oklch(0.65 0.02 260);

  /* Lines / focus */
  --border:        oklch(0.82 0.008 80);
  --border-strong: oklch(0.72 0.012 80);
  --ring:          oklch(0.6 0.15 255);

  /* Accent */
  --accent:           oklch(0.58 0.19 255);
  --accent-fg:        oklch(0.99 0 0);
  --accent-soft:      oklch(0.92 0.04 255);
  --accent-soft-fg:   oklch(0.35 0.18 255);

  /* Semantic */
  --danger:    oklch(0.6 0.22 25);
  --success:   oklch(0.62 0.15 155);
  --warning:   oklch(0.72 0.15 75);

  /* Overlay surface */
  --popover:        oklch(0.99 0.003 80);
  --popover-fg:     var(--fg);
  --popover-border: oklch(0.85 0.008 80);

  /* Elevation */
  --shadow-sm:  0 1px 2px oklch(0.15 0.02 260 / 0.06);
  --shadow-md:  0 4px 12px oklch(0.15 0.02 260 / 0.10), 0 1px 2px oklch(0.15 0.02 260 / 0.06);
  --shadow-lg:  0 12px 32px -8px oklch(0.15 0.02 260 / 0.18), 0 4px 8px oklch(0.15 0.02 260 / 0.08);

  /* Radii */
  --radius-sm: 6px;
  --radius-md: 8px;
  --radius-lg: 12px;
  --radius-xl: 16px;
}
```

The `.dark` selector contains the same tokens with dark-mode values. Both blocks
ship in `@mxv/ui/globals.css`.

## Change the accent color

Most projects only ever touch this:

```css
:root {
  --accent:        oklch(0.65 0.21 145);  /* green-ish */
  --accent-fg:     oklch(0.99 0 0);
  --accent-soft:   oklch(0.92 0.08 145);
  --accent-soft-fg: oklch(0.30 0.18 145);
}
```

Every `<Button tone="primary">`, focus ring, and accent surface follows.

## Why oklch?

When you tweak lightness to build hover or active states, oklch gives you
perceptually-uniform stops across hues. The same `+0.05` lightness shift on a
red and a blue feels equally bright. HSL can't do that.

If you'd rather author in hex/HSL, convert at build time — the runtime values
are CSS variables, the library doesn't care what color space you authored in.

## Per-component overrides

Every component accepts a `className`. Use it for one-off tweaks:

```tsx
<Button className="rounded-full px-6">Pill</Button>
```

For sustained per-component changes, write your own wrapper component instead
of fighting the className cascade.
